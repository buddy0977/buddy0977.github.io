EXP 1Ultrasonic & Buzzer
// Ultrasonic Sensor Pins
const int trigPin = 9;
const int echoPin = 10;
// Buzzer Pin
const int buzzer = 8;
long duration;
int distance;

void setup() {
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
  pinMode(buzzer, OUTPUT);
  Serial.begin(9600);
}

void loop() {
  // Clear trigger pin
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);
  // Send ultrasonic pulse
  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);
    
  // Read echo pulse
  duration = pulseIn(echoPin, HIGH);
  // Calculate distance
  distance = duration * 0.034 / 2;
  // Display distance
  Serial.print("Distance: ");
  Serial.print(distance);
  Serial.println(" cm");
    
  // Buzzer condition
  if(distance < 25) {
    digitalWrite(buzzer, HIGH);
  }
  else {
    digitalWrite(buzzer, LOW);
  }
  delay(200);
}
HC-SR04 Passive Buzzer Arduino Uno
Overview

The HC-SR04 ultrasonic sensor measures distance by emitting a 40 kHz sound pulse from its TRIG pin and timing how long the echo takes to return to the ECHO pin. Distance is calculated using the formula: distance = (duration × 0.034) / 2, where 0.034 cm/µs is the speed of sound and dividing by 2 accounts for the round trip.

The buzzer acts as an alert — when the measured distance falls below 25 cm, it activates to signal proximity. This is the fundamental principle behind parking sensors and obstacle-avoidance systems.

pulseIn() is a built-in Arduino function that measures the duration (in microseconds) of a HIGH pulse on a given pin. The TRIG pin must be pulsed HIGH for at least 10 µs to trigger a measurement cycle.

16x2 LCD LiquidCrystal Lib Arduino Uno
Overview

The 16x2 LCD (Liquid Crystal Display) can show 16 characters across 2 rows. It is driven using the Hitachi HD44780 controller, which the LiquidCrystal library abstracts into simple function calls.

It is wired in 4-bit mode here — only D4–D7 data pins are used instead of all 8, which saves GPIO pins without sacrificing functionality. RS (Register Select) chooses between command and data mode, and EN (Enable) latches each instruction into the display.

lcd.begin(16, 2) initialises the display size. lcd.setCursor(col, row) positions the text cursor before printing. Row and column indices start at 0, so setCursor(0,1) moves to the beginning of the second line.

  // Stop Motor

  digitalWrite(IN2, LOW);
  delay(1000);
    
  // Motor Reverse Direction
  digitalWrite(IN1, LOW);
  digitalWrite(IN2, HIGH);

  // Stop Motor
  digitalWrite(IN1, LOW);

}
L298N Driver DC Motor Arduino Uno
Overview

A DC motor cannot be driven directly from an Arduino GPIO pin — the pin can only supply ~40 mA, far less than what a motor needs. The L298N is an H-bridge motor driver IC that can supply up to 2 A per channel, powered by an external supply.

The H-bridge works by controlling two pairs of switches. Setting IN1=HIGH and IN2=LOW drives current in one direction (forward); reversing them (IN1=LOW, IN2=HIGH) reverses current flow, spinning the motor backward. Setting both LOW coasts or brakes the motor.

ENA (Enable A) controls speed via PWM (Pulse Width Modulation). analogWrite() outputs a 0–255 value representing 0–100% duty cycle. A value of 200 is ~78% speed, and 150 is ~59% speed, allowing smooth variable-speed control without wasting power as heat.

DHT11 DHT11.h Library Passive Buzzer Arduino Uno
Overview

The DHT11 is a low-cost digital temperature and humidity sensor. It uses a single-wire protocol to send calibrated data as a 40-bit serial stream. The DHT11.h library simplifies reading to a single call: readTemperature(), which returns an integer Celsius value.

The DHT11 measures temperatures from 0–50°C with ±2°C accuracy and humidity from 20–90% RH. It requires a pull-up resistor (typically 10 kΩ) on the data line, though many breakout modules include this on-board.

Here the buzzer activates when temperature drops below 25°C — an inverted threshold used as a cold-alert rather than an overheat alarm. This variant uses the simpler DHT11.h wrapper as compared to the full-featured DHT.h library used in Exp 4B.

EXP 4BDHT11 & Buzzer (DHT.h)
#include <DHT.h>

// DHT11 setup
#define DHTPIN 2
#define DHTTYPE DHT11
DHT dht(DHTPIN, DHTTYPE);
// Buzzer pin
const int buzzer = 8;
// Temperature threshold
float threshold = 30.0;

void setup() {
  Serial.begin(9600);
  dht.begin();
  pinMode(buzzer, OUTPUT);
}

void loop() {
  // Read temperature
  float temperature = dht.readTemperature();
  // Display temperature
  Serial.print("Temperature: ");
  Serial.print(temperature);
  Serial.println(" °C");
    
  // Check threshold
  if (temperature > threshold) {
    digitalWrite(buzzer, HIGH);
  }
  else {
    digitalWrite(buzzer, LOW);
  }
  delay(2000);
}
DHT11 DHT.h Library Passive Buzzer Arduino Uno
Overview

This variant uses the Adafruit DHT unified library (DHT.h), which supports DHT11, DHT22, and DHT21 through a common interface selected via the DHTTYPE macro. It is more widely used and maintained than the simpler DHT11.h wrapper.

readTemperature() returns a float rather than an integer, giving finer decimal resolution. The threshold is set at 30.0°C — the buzzer fires when exceeded, making this a classic overheat alert applicable to server rooms, battery packs, or greenhouse monitoring.

A 2-second delay between readings is important: the DHT11 has a minimum sampling period of 1 second. Polling faster can return stale or incorrect values due to the sensor's internal capacitor charge cycle not completing in time.

EXP 5LPC2148 – Stepper Motor
#include <lpc21XX.h>

void delay()
{
  int i,j;
  for(i=0;i<1000;i++)
  {
    for(j=0;j<1000;j++);
  }
}

void main(void)
{
  IODIR0 = 0X0000FFFF;
  while(1)
  {
    IOSET0 = 0X00000914;
    delay();
    IOCLR0 = 0X0000FFFF;
    IOSET0 = 0X00000A21;
    delay();
    IOCLR0 = 0X0000FFFF;
    IOSET0 = 0X00001409;
    delay();
    IOCLR0 = 0X0000FFFF;
    IOSET0 = 0X0000210A;
    delay();
    IOCLR0 = 0X0000FFFF;
  }
}
LPC2148 Stepper Motor ARM7TDMI-S lpc21XX.h
Overview

The LPC2148 is a 32-bit ARM7TDMI-S based microcontroller by NXP. Unlike Arduino, it is programmed in bare-metal C using memory-mapped I/O registers. IODIR0 configures the direction of Port 0 pins — writing 0x0000FFFF sets the lower 16 pins as outputs.

IOSET0 and IOCLR0 are write-only registers that set or clear specific Port 0 pins without affecting others. Each hex value written to IOSET0 represents a specific bit pattern that energises a combination of stepper motor coils for one step in the sequence.

A stepper motor moves in discrete angular steps by energising coils in a defined sequence. The four hex values (0x0914, 0x0A21, 0x1409, 0x210A) represent a full-step drive pattern — each activating two coils at a time. Cycling through these four patterns in order rotates the motor continuously. The nested-loop software delay controls step speed without any hardware timer.

EXP 6Rate Monotonic Scheduling
#include <iostream>
#include <math.h>
using namespace std;

int main() {
  int a, temp, i, j;
  float ut = 0;
  float u[10], t[10], c[10], p[10];
  cout << "Enter the number of tasks: ";
  cin >> a;
    
  for (i = 0; i < a; i++) {
    cout << "Enter period of task " << i + 1 << ": ";
    cin >> t[i];
    p[i] = t[i];
    cout << "Enter CPU time of task " << i + 1 << ": ";
    cin >> c[i];
  }
    
  for (i = 0; i < a - 1; i++) {
    for (j = 0; j < a - i - 1; j++) {
      if (p[j] > p[j + 1]) {
        temp = p[j];
        p[j] = p[j + 1];
        p[j + 1] = temp;
      }
    }
  }
    
  cout << "\nTask Priorities:\n";
  for (i = 0; i < a; i++) {
    for (j = 0; j < a; j++) {
      if (t[i] == p[j]) {
        cout << "Priority of Task " << i + 1 << " is: " << j + 1 << endl;
      }
    }
  }
    
  for (i = 0; i < a; i++) {
    u[i] = c[i] / t[i];
    ut += u[i];
  }
  cout << "\nTotal Utilization is: " << ut << endl;
  if (ut < 1) {
    cout << "Tasks are Schedulable" << endl;
  } else {
    cout << "Tasks are NOT Schedulable" << endl;
  }
  return 0;
}
Rate Monotonic RTOS Scheduling C++ / Simulation
Overview

Rate Monotonic Scheduling (RMS) is a fixed-priority preemptive scheduling algorithm used in real-time systems. The core rule is simple: tasks with shorter periods are assigned higher priorities. This is optimal among all fixed-priority algorithms — if any fixed-priority assignment can meet all deadlines, RMS will too.

This program implements RMS analysis by taking each task's period (T) and CPU execution time (C), then computing CPU utilization as U = C/T per task. Total utilization is the sum of all individual utilizations. The schedulability condition checked here is U < 1 (necessary condition). The tighter Liu & Layland bound is U ≤ n(2^(1/n) − 1), which approaches ln(2) ≈ 69.3% as n grows.

Priority assignment is done by sorting periods in ascending order using bubble sort — the task with the smallest period gets priority rank 1 (highest). The program then maps each original task back to its sorted position to display its RMS priority. This simulation is useful for verifying task sets before deploying them on an embedded RTOS like FreeRTOS.

EXP 7RTOS – FreeRTOS LED Tasks
#include <Arduino_FreeRTOS.h>

// LED Pins
const int redLED = 8;
const int greenLED = 9;
// Task Functions
void RedLEDTask(void *pvParameters);
void GreenLEDTask(void *pvParameters);

void setup() {
  pinMode(redLED, OUTPUT);
  pinMode(greenLED, OUTPUT);
  // Create Tasks
  xTaskCreate(
    RedLEDTask,
    "Red LED Task",
    128,
    NULL,
    1,
    NULL
  );
  xTaskCreate(
    GreenLEDTask,
    "Green LED Task",
    128,
    NULL,
    1,
    NULL
  );
}

void loop() {
  // Empty because FreeRTOS scheduler handles tasks
}

// Task for Red LED
void RedLEDTask(void *pvParameters) {
  while (1) {
    digitalWrite(redLED, HIGH);
    vTaskDelay(500 / portTICK_PERIOD_MS);
    digitalWrite(redLED, LOW);
    vTaskDelay(500 / portTICK_PERIOD_MS);
  }
}

// Task for Green LED
void GreenLEDTask(void *pvParameters) {
  while (1) {
    digitalWrite(greenLED, HIGH);
    vTaskDelay(1000 / portTICK_PERIOD_MS);
    digitalWrite(greenLED, LOW);
    vTaskDelay(1000 / portTICK_PERIOD_MS);
  }
}
FreeRTOS Red LED Green LED Arduino Uno
Overview

FreeRTOS (Free Real-Time Operating System) is a lightweight kernel that enables multitasking on microcontrollers. Unlike standard Arduino sketches where loop() executes sequentially, FreeRTOS allows multiple tasks to run concurrently by rapidly switching between them — a process called context switching.

xTaskCreate() registers a task with the scheduler. Parameters include the function pointer, a name string, stack size in words (128 here), optional parameters, priority level, and a handle. Both tasks have equal priority (1), so the scheduler round-robins between them using time slices.

vTaskDelay() is used instead of delay() — it yields CPU control back to the scheduler during the wait period, allowing other tasks to run. portTICK_PERIOD_MS converts milliseconds to RTOS ticks. The red LED blinks every 500 ms and the green every 1000 ms, both running independently without blocking each other.
