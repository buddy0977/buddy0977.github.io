Family Tree — Prolog

Prolog 
/* Facts */
parent_of(helen, lily).
parent_of(oliver, james).
parent_of(sophie, james).
parent_of(jess, simon).
parent_of(ali, simon).
parent_of(lily, harry).
parent_of(james, harry).

/* Rules */
father_of(X,Y) :- male(X), parent_of(X,Y).
mother_of(X,Y) :- female(X), parent_of(X,Y).

grandfather_of(X,Y) :- male(X), parent_of(X,Z), parent_of(Z,Y).
grandmother_of(X,Y) :- female(X), parent_of(X,Z), parent_of(Z,Y).

sister_of(X,Y) :-
    female(X),
    father_of(F,Y), father_of(F,X), X \= Y.
sister_of(X,Y) :- female(X),
    mother_of(M,Y), mother_of(M,X), X \= Y.

aunt_of(X,Y) :- female(X),
    parent_of(Z,Y), sister_of(Z,X), !.

brother_of(X,Y) :-
    male(X),
    father_of(F,Y), father_of(F,X), X \= Y.
brother_of(X,Y) :- male(X),
    mother_of(M,Y), mother_of(M,X), X \= Y.

uncle_of(X,Y) :-
    parent_of(Z,Y), brother_of(Z,X).

ancestor_of(X,Y) :- parent_of(X,Y).
ancestor_of(X,Y) :- parent_of(X,Z), ancestor_of(Z,Y).
// Sample Queries
Q1
?- parent_of(lily, harry).→ true. (Lily is a parent of Harry)
Q2
?- grandfather_of(oliver, harry).→ true. (Oliver → James → Harry)
Q3
?- grandmother_of(X, harry).→ X = helen ; X = sophie. (All grandmothers of Harry)
Q4
?- ancestor_of(helen, harry).→ true. (Helen is ancestor of Harry through Lily)
Q5
?- uncle_of(X, harry).→ X = simon. (James's brother Simon is uncle of Harry)
Theory Family Tree — Key Concepts
  • 01Prolog is a logic programming language based on formal logic. A program consists of facts (ground truths) and rules (logical implications) that are queried using a built-in inference engine via backward chaining.
  • 02Facts like parent_of(helen, lily) assert unconditional truths. Rules use the :- operator (read as "if") to define relationships derived from other facts or rules.
  • 03Variables in Prolog are uppercase (e.g., X, Y, Z) and are unified (bound) during resolution. Atoms are lowercase constants like helen, lily.
  • 04Recursive rules such as ancestor_of use Prolog's built-in depth-first search to traverse chains of relationships. The base case terminates recursion when parent_of(X,Y) holds directly.
  • 05The cut operator ! in aunt_of prunes the search tree — once a valid aunt is found, Prolog stops exploring alternative solutions, preventing duplicate results.
  • 06The \= constraint ensures inequality, preventing a person from being their own sibling (e.g., X \= Y in sister_of).
  • 07Prolog uses backtracking to find all solutions. Querying grandmother_of(X, harry) causes Prolog to unify and explore all possible bindings of X that satisfy the rule.
  • 08Multiple clauses for the same predicate (e.g., two clauses for sister_of) represent disjunction — a sister through either the father's or mother's side satisfies the predicate.

Breadth-First Search — Prolog

Prolog 
%connected(+Start, +Goal, -Weight)
connected(1,7,1).   connected(1,8,1).   connected(1,3,1).
connected(7,4,1).   connected(7,20,1).  connected(7,17,1).
connected(8,6,1).   connected(3,9,1).   connected(3,12,1).
connected(9,19,1).  connected(4,42,1).  connected(20,28,1).
connected(17,10,1).

connected2(X,Y,D) :- connected(X,Y,D).
connected2(X,Y,D) :- connected(Y,X,D).

next_node(Current, Next, Path) :-
    connected2(Current, Next, _),
    not(member(Next, Path)).

breadth_first(Goal, Goal, _, [Goal]).
breadth_first(Start, Goal, Visited, Path) :-
    findall(X,
        (connected2(X,Start,_), not(member(X,Visited))),
        [T|Extend]),
    write(Visited), nl,
    append(Visited, [T|Extend], Visited2),
    append(Path, [T|Extend], [Next|Path2]),
    breadth_first(Next, Goal, Visited2, Path2).
// Sample Queries
Q1
?- breadth_first(1, 28, [1], Path).→ Finds path from node 1 to node 28 via BFS
Q2
?- connected2(7, X, _).→ X = 1 ; X = 4 ; X = 20 ; X = 17. (All neighbours of 7)
Q3
?- breadth_first(1, 42, [1], Path).→ Path goes 1 → 7 → 4 → 42
Q4
?- next_node(3, X, [1,3]).→ X = 9 ; X = 12. (Unvisited neighbours of 3)
Q5
?- breadth_first(1, 19, [1], Path).→ Path: 1 → 3 → 9 → 19
Theory BFS — Key Concepts
  • 01Breadth-First Search (BFS) explores a graph level by level — all nodes at depth 1 are visited before any node at depth 2. This guarantees finding the shortest path in an unweighted graph.
  • 02connected2 makes the graph bidirectional by allowing traversal in both directions, even though the original connected facts are one-directional.
  • 03findall/3 collects all unvisited neighbours of the current node into a list, simulating the BFS queue expansion step.
  • 04The Visited list prevents revisiting already-explored nodes, avoiding infinite loops in cyclic graphs and ensuring each node is processed at most once.
  • 05The base case breadth_first(Goal, Goal, _, [Goal]) terminates recursion when the current frontier node matches the goal, returning it as a single-element path.
  • 06BFS uses more memory than DFS (O(b^d) space where b=branching factor, d=depth) because it maintains the entire frontier queue, but guarantees optimality for uniform-cost edges.
  • 07append/3 is used to manage both the visited list and the path — merging the current visited set with new neighbours and updating the traversal queue simultaneously.

Depth-First Search — Prolog

Prolog 
solve(Node, Solution) :-
    depthfirst([], Node, Solution).

depthfirst(Path, Node, [Node | Path]) :-
    goal(Node).
depthfirst(Path, Node, Sol) :-
    s(Node, Node1),
    \+ member(Node1, Path),
    depthfirst([Node | Path], Node1, Sol).

depthfirst2(Node, [Node], _) :-
    goal(Node).
depthfirst2(Node, [Node | Sol], Maxdepth) :-
    Maxdepth > 0,
    s(Node, Node1),
    Max1 is Maxdepth - 1,
    depthfirst2(Node1, Sol, Max1).

/* Goals and State-Space */
goal(f).    goal(j).
s(a,b).    s(a,c).
s(b,d).    s(b,e).
s(c,f).    s(c,g).
s(d,h).    s(e,i).    s(e,j).
// Sample Queries
Q1
?- solve(a, Solution).→ Solution = [f, c, a] (first path DFS finds to a goal)
Q2
?- depthfirst2(a, Path, 3).→ Path = [a, b, d, h] or [a, c, f] (depth-limited)
Q3
?- goal(X).→ X = f ; X = j. (All goal states)
Q4
?- depthfirst([], a, Sol).→ Sol = [f, c, a] (same as solve/2)
Q5
?- depthfirst2(a, Path, 4).→ Path = [a, b, e, j] (finds second goal j within depth 4)
Theory DFS — Key Concepts
  • 01Depth-First Search (DFS) explores as deep as possible along each branch before backtracking. In Prolog, this maps naturally to the language's own recursive + backtracking execution model.
  • 02The Path accumulator carries visited nodes to prevent cycles. The check \+ member(Node1, Path) ensures we never re-enter a previously explored state.
  • 03The solve/2 wrapper initialises the visited path as empty ([]) and delegates to depthfirst/3, separating interface from implementation.
  • 04DFS has O(b·m) space complexity (b = branching factor, m = max depth) — far more memory-efficient than BFS for deep trees, though it may not find the shortest path.
  • 05Depth-Limited DFS (depthfirst2) adds a Maxdepth parameter that decrements with each recursive call, stopping exploration once the limit is reached — useful for iterative deepening.
  • 06The state-space graph is defined by s/2 (successor relation), making the search algorithm cleanly independent of the problem — different problems only require different s/2 and goal/1 definitions.
  • 07DFS can get trapped exploring infinite paths in graphs without a depth limit or visited-node check. Both protective mechanisms are demonstrated here — cyclic check in depthfirst and depth cap in depthfirst2.

Hill Climbing — Python

Python 
def f(x):
    return -x*x + 5

x = 2.0
step = 0.1
iterations = 0

for i in range(100):
    left  = x - step
    right = x + step
    if f(left) > f(x):
        x = left
        iterations += 1
    elif f(right) > f(x):
        x = right
        iterations += 1
    else:
        print(f"Algorithm converged after {iterations} iterations.")
        break

if abs(x) < 1e-10:
    x = 0.0

print("Best solution =", x)
print("Maximum value =", round(f(x), 4))
Theory Hill Climbing — Key Concepts
  • 01Hill Climbing is a local search algorithm that continuously moves to the neighbouring state with the highest value (greedy best neighbour). It is an iterative improvement technique with no memory of previously visited states.
  • 02The objective function here is f(x) = -x² + 5, a downward parabola with a global maximum at x = 0 where f(0) = 5. The negative quadratic ensures a single peak — an ideal test case.
  • 03At each iteration, two neighbours are evaluated: x - step (left) and x + step (right). The algorithm moves to whichever neighbour improves the objective; if neither does, it terminates.
  • 04The step size (0.1) controls the resolution of the search. Larger steps speed up convergence but may skip over the exact peak; smaller steps are more precise but slower.
  • 05Hill Climbing can get stuck at local maxima, plateaus, or ridges — points where no immediate neighbour improves the objective but the global optimum hasn't been reached. This is its primary limitation.
  • 06Convergence is detected when neither left nor right neighbour yields improvement, meaning the algorithm has reached a local (and in this case, global) optimum. The loop then breaks early.
  • 07The near-zero check (abs(x) < 1e-10) handles floating-point precision — since the true optimum is x = 0, values extremely close to zero are snapped to exactly 0.0 for clean output.
  • 08Variants like Stochastic Hill Climbing, Random Restarts, and Simulated Annealing address the local-maxima problem by introducing randomness or occasional downhill moves to escape suboptimal regions.

Genetic Algorithm — Python

Python 
import random

TARGET = 20

# Fitness Function
def fitness(x):
    return -abs(TARGET - x)

# Initial Population
population = [random.randint(0, 30) for _ in range(6)]

for generation in range(10):
    # Selection (best 2 individuals)
    population.sort(key=fitness, reverse=True)
    parent1 = population[0]
    parent2 = population[1]

    # Crossover
    child = (parent1 + parent2) // 2

    # Mutation
    if random.random() < 0.3:
        child += random.choice([-1, 1])

    # Replace worst individual
    population[-1] = child
    print("Generation", generation + 1, "Best =", parent1)

print("\nBest Solution =", population[0])
Theory Genetic Algorithm — Key Concepts
  • 01Genetic Algorithms (GAs) are population-based metaheuristic optimisation algorithms inspired by Darwinian evolution — survival of the fittest, crossover, and mutation drive the population toward better solutions over generations.
  • 02The fitness function -abs(TARGET - x) assigns higher fitness to individuals closer to the target value (20). A fitness of 0 means a perfect solution; increasingly negative values indicate increasing distance from the target.
  • 03Selection identifies the two fittest individuals (parents) from the current population by sorting using the fitness key. This is a simple truncation selection strategy — only the top 2 reproduce.
  • 04Crossover (recombination) creates a child by averaging the two parent values: (parent1 + parent2) // 2. This mimics biological crossover where offspring inherit traits from both parents.
  • 05Mutation introduces randomness by randomly adding or subtracting 1 from the child with 30% probability. This prevents the population from converging prematurely and maintains genetic diversity.
  • 06Elitism is implicit — the worst individual is replaced by the new child while the best individuals are preserved across generations, ensuring the population quality never decreases.
  • 07GAs are particularly powerful for large, complex search spaces where gradient information is unavailable. Unlike hill climbing, they maintain a diverse population, reducing the risk of getting trapped at local optima.
  • 08The key parameters — population size (6), mutation rate (0.3), and number of generations (10) — control the exploration-exploitation trade-off. Larger populations and more generations improve solution quality at the cost of computation time.
